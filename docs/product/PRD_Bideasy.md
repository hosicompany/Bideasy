## **1\. 제품 개요 (Product Overview)**

### **1.1 배경 및 목적**

기존 입찰 시장의 불필요한 '공포 마케팅(낙찰 예측)'을 지양하고, **데이터 기반의 '관리'와 '분석'** 가치를 제공합니다.

### **1.2 핵심 가치 (Core Values)**

* **Radical Simplicity (단순함):** 복잡한 필터 제거, Zero UI.  
* **Invisible Safety (안전함):** 시스템이 적자 수주 원천 차단.  
* **Intellectual Honesty (정직함):** **\[New\]** 무책임한 '낙찰가 찍기(Prediction)'를 제공하지 않으며, AI는 사용자의 판단을 돕는 '비서' 역할에 집중합니다.

### **1.3 AI 도입 전략 (AI Strategy)**

* **Do (할 것):** 복잡한 공고문의 **요약**, 독소 조항(위험 조건) **탐지**, 과거 데이터 기반의 **팩트 분석**.  
* **Don't (안 할 것):** 로또 번호 맞추기 식의 **낙찰가 예측**. (사용자 신뢰 보호)

## **3\. 핵심 기능 명세 (Functional Specifications)**

### **\[P0\] F-01. 제로 UI 홈 (Smart Discovery Feed)**

* **기능:** 사용자(면허/지역) 맞춤 공고 자동 큐레이션.  
* **UX:** "사장님, 오늘 2건 있어요\!" (카드 뉴스 형태).  
* **Empty State:** 조건에 맞는 공고가 없을 경우 감성 문구 노출.

### **\[P0\] F-02. 게이미피케이션 투찰가 계산기**

* **기능:** 슬라이더(-2% \~ \+2%)를 통한 직관적 사정률 조절.  
* **Safety Net:** 낙찰하한가 미만 진입 시 UI 전체 **Red Alert** 전환 및 복사 버튼 비활성화.

### **\[P0\] F-03. Pay-per-Bid 과금 시스템**

* **과금 시점:** 계산된 투찰 금액을 **\[복사하기\]** 또는 **\[저장하기\]** 버튼을 누르는 순간.  
* **결제 수단:** 인앱 포인트 또는 보상형 광고 시청.

### **\[Phase 2\] F-04. AI 공고 분석 비서 (Smart Briefing)**

* **기능:** LLM(거대언어모델)을 활용한 공고문 텍스트 분석.  
* **Output:**  
  1. **3줄 요약:** 공사 개요, 필수 자격, 마감일.  
  2. **Red Flag 탐지:** "착공 후 30일 내 완공", "특수 실적 필요" 등 위험 조건 하이라이팅.  
* **가치:** 모바일에서 긴 HWP 파일을 열어보는 불편함 해소.

## **4\. 기술적 요구사항 (Technical Requirements)**

### **4.1 클라이언트 & 서버**

* **Frontend:** Flutter (Cross Platform).  
* **Backend:** Python FastAPI.  
* **AI Engine (Phase 2):** OpenAI API (GPT-4o mini) 또는 Local LLM 활용. 공고문 텍스트 파싱 및 요약 프롬프트 엔지니어링 필요.

### **4.2 데이터 파이프라인**

* **공공데이터포털 API:** 입찰공고정보서비스 활용.  
* **Resilience:** API 장애 시 크롤링(Scraping) 모듈 자동 전환.

## **5\. 성공 지표 (Success Metrics)**

1. **Safety Catch:** 시스템에 의한 무효 입찰 방지 횟수.  
2. **Retention:** D+30 재방문율 40% 이상.  
3. **AI Usage (Phase 2):** 공고 상세 조회 대비 'AI 요약 보기' 클릭률 50% 이상.
