# BidEasy 콘텐츠 발행 캘린더 (3개월)

> 확정일 2026-06-16 · OSMU 3버킷 + **신선도별 트랙** 분리
> 상록수(A)는 *주제*를 미리 잡는다(이 문서). 데이터스토리(B)는 *정기 슬롯*만 잡고 **내용은 발행 시점 최신 데이터로 자동 생성**. 소식(C)은 이벤트 발생 시 인터럽트.

## 운영 원칙
- **주 2~3편.** 신선도 = B(주간 데이터스토리)가, 검색 유입 토대 = A(상록수)가 책임.
- 모든 글: 계산기·14일 체험 **CTA** + **내부링크** + SSR/OG/JSON-LD/sitemap(기존 패턴 재사용).
- 발행 = **AI 초안 → 사람 검수(법·숫자 정확성)** → 발행. 완전 무인 금지(브랜드=정밀).
- 끝에 항상 **bideasy.kr로 트래픽 회수**.

## Track B — 정기 데이터스토리 포맷 (잠금) ⭐

> 모두 우리 개찰 데이터(`OpeningResult`: `participants_count`·`winner_rate`·`winner_price`) 기반 = **경쟁사 복제 불가 + 자동 생성 최적**. 발행 시점 데이터로 매번 새로 생성.

| 슬롯 | 포맷 | 데이터 소스 | 카드뉴스 |
|---|---|---|---|
| 주간(월) | 지난주 **경쟁률 TOP10** | `participants_count` 내림차순 | ◎ 랭킹·숫자 최적 |
| 주간(금) | 이번 주 **단독응찰·저경쟁 기회 공고** | `participants_count` ≤ N | ◎ 역발상 후킹 |
| 월간(1주차) | **○월 입찰 동향 리포트** | 공종별 `winner_rate` 평균·물량 | ○ |

각 글 URL은 **날짜 슬러그**(예: `/blog/weekly-2026-w25`)로 — 공유·신선도 신호에 유리.
